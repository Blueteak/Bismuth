# Bismuth Visualizer Plan

## Objective

Stateless browser experience: visibly grow one plausible lab-grown bismuth
hopper in real time; GPU surface extraction; age-driven metallic iridescence.

## Current gate: target morphology

Milestone 1B blocks presentation. Infrastructure exists: generic cubic solver,
live GPU extraction, controller, initial material.

Candidate 2C: product-rejected. Regular six-support, complete-loop `{1-102}`
carrier derives from microscopic polycrystalline Sn-Bi + project geometry, not
bulk targets. Keep its exact ledger/WebGPU seam as mechanism evidence only.

Candidate 2D: active, target-locked. Authority: `crystal_references/` + matching
bulk elemental-Bi evidence. Mismatched sources may support declared isolated
mechanisms/numerics only; never geometry, facets, coefficients, acceptance.

Closed evidence:

- Prescribed partial-front carrier: topology, volume/energy, refinement,
  changing production meshes pass; shallow repeated terrace habit fails.

  one-front 3D extraction pass; strip morphology and persistent supply fail.
  No sourced recurrence/winding law; no target-source promotion.

Edge/free-surface isolation now passes one real seeded contact, signed Stefan
driver, exact ledger, null/reversal, scalar, changing WebGPU meshes. Result is
another ribbon; persistent supply, route selection, target morphology fail.

Next: Candidate 2D strategy review. Do not blend/tune closed local mechanisms.
Seek target-matched bulk-Bi evidence for persistent source + path selection. If

## Milestones

### Foundation - complete

React/Vite/TS, Express, honest WebGPU failure, HDRI, controller boundary.

### Generic numerical scaffold - complete

Published cubic phase field: deterministic faceted-hopper regression only.
Fractal/dendritic non-reproduction is closed; not an acceptance gate.

### GPU extraction/live integration - complete infrastructure

Classification, compaction, vertices/normals/age, bounded promotion, indirect
draw, live controller. No full-volume production readback.

### Target-locked bismuth hopper - current

Exit:

- Every morphology claim passes specimen-match audit.
- One initial site -> connected deep rectilinear/rhombohedral-pyramidal hopper
  with continuous many-elevation winding ledges.
- Approximate right-angle projected turns; natural uneven spacing,
  persistence, center; partial/interrupted fronts.
- Deterministic non-self-intersecting paths; exact swept-volume + latent ledger.
- Reject regular hexagon, three-sided Sn-Bi pyramid, homothetic rings, smooth
  bowl, generic cubic hopper.
- Single sector passes references 1-2 before connected branching/intergrowth
  against 3-4.
- Selected source/thermal driver closes balance; stable across frozen schedules.
- Extraction preserves volume, connectivity, topology, capacity, sensible grid
  refinement; never feeds back into growth.
- No acceptance via decorative/postprocessed geometry.
- Every scientific slice, pass or fail: frozen deterministic integrated WebGPU
  3D result + fixed-camera screenshot beside all four references; direct pixel
  inspection + explicit visual verdict. Show actual state; no retired carrier,
  fallback, camera/material retune, decoration, metrics-only closeout.
- Compact state visibly changes production extraction/indirect-draw meshes at
  production-plausible cadence. CPU full-volume upload: development proof only.
- Several deterministic source schedules preserve deep-hopper character before
  coefficient promotion.

### Material/camera/public lifecycle - next

Only after morphology. Calibrate oxide, metal, environment, fixed camera; then
public auto-start, `Stop`, completion, `Regenerate`, disposal, device loss.

### Multi-orientation specimen - later

Require documented multiphase/orientation fields + shared transport. No
overlapping cosmetic meshes.

### Performance selection - later

Choose grid/cadence from accepted model. Benchmark only for that decision;
present fidelity/duration/memory/update-rate tradeoffs before equation or
resolution changes.

### Deployment - later

Vite build via stateless Express on Ubuntu EC2 behind trusted HTTPS. No DB,
queue, sessions, auth, or server solver.
