# Bismuth Visualizer Plan

## Objective

Stateless browser experience: visibly grow one plausible lab-grown bismuth
hopper in real time; GPU surface extraction; age-driven metallic iridescence.

## Current gate: target morphology

Milestone 1B blocks presentation. Infrastructure exists: generic cubic solver,
live GPU extraction, controller, initial material.

Candidate 2C: product-rejected. Regular six-support, complete-loop `{1-102}`
carrier derives from microscopic polycrystalline Sn-Bi + project geometry, not
bulk targets. Keep its exact ledger/WebGPU seam as mechanism evidence only.

Candidate 2D: retired. Scripted planar paths, rails, treads, and risers passed
numerics/extraction but repeatedly failed bulk 3D morphology. Code removed.

Retired candidate scope/approach/results: `docs/simulation-model.md`. Their
code, tests, and review routes are removed; no historical fixture remains active.

Candidate 2E: active. Orientation-aware 3D cellular growth. CA is numerical
representation, not a claimed Bi mechanism. Tests 5-12 reject broad-interface
attachment as the body carrier. Active hypothesis: dynamic surface-layer/ledge
fronts build the body by lateral sweeps in each seed-local frame. Authority:
`crystal_references/`; source boundary: `docs/references.md`.

Current decision: sparse edge-source Test 1 proves repeated sources can make
finite-depth walls while preserving an opening, but independent per-site births
scale with exposed area and collapse into a smooth centered block. Stop. Do not
tune period/probability. Next strategy must define a stateful, refinement-aware
source population; no transport or physical recurrence claim.

- One shared GPU storage grid; authoritative phase/fill/supply/age plus dynamic
  surface-layer/front state.
- Per-seed owner + local morphology-frame quaternion; no global habit alignment.
- Shared supply; synchronous capture; deterministic arrival-time impingement;
  no iteration-order ownership or overlapping meshes.
- No prepainted paths, target masks, special empty cells, stamped ledges.
- One-seed local sweep/handoff proven; finite-volume body carrier unresolved.

### Target-locked bismuth hopper - current

Exit:

- Every morphology claim passes specimen-match audit.
- 2E.1 reset: one seed; dynamic interface-generated layer/ledge fronts are the
  primary capture operator. Lateral sweeps must build connected bulk outer
  planes, a deep recess, and visible winding; references 1-2. Broad faces do
  not capture everywhere; no spatially authored route.
- 2E.1 Test 1: isolated terrace stall + one coherent layer passed; integrated
  front built one plane but retained a cube/slab with no recess or winding.
  Closed target fail.
- 2E.1 Test 2: one conserved source reached four connected lateral planes and
  left the center liquid; integrated result was a thin box wall, not bulk
  planes. Closed target fail.
- 2E.2 Test 1: frozen outer-edge trials made finite depth/open center but grew
  563 sources by integrated step 96 and merged into a smooth centered block.
  Closed sparsity/scaling/target fail.
- 2E.2 later: front persistence/termination/merging and supply coupling ->
  uneven widths, offsets, interruptions, and asymmetry.
- 2E.3: arbitrary seed frames + shared transport + deterministic impingement;
  references 3-5. Same local rule per frame.
- Approximate right-angle projected turns; natural uneven spacing,
  persistence, center; partial/interrupted fronts.
- Exact supply/solid/latent ledger; connected monotonic solidification.
- Reject regular hexagon, three-sided Sn-Bi pyramid, homothetic rings, smooth
  bowl, generic cubic hopper.
- Rotation-equivariant output across frozen awkward seed/grid angles; grid and
  time refinement; collision result independent of seed ID/update order.
- Selected source/thermal driver closes balance; stable across frozen schedules.
- Extraction preserves volume, connectivity, topology, capacity, sensible grid
  refinement; never feeds back into growth.
- No acceptance via decorative/postprocessed geometry.
- Every scientific slice, pass or fail: frozen deterministic integrated WebGPU
  3D result + fixed-camera screenshot beside all five references; direct pixel
  inspection + explicit visual verdict. Show actual state; no retired carrier,
  fallback, camera/material retune, decoration, metrics-only closeout.
- GPU CA state visibly changes production extraction/indirect-draw meshes at
  production-plausible cadence. No production CPU full-volume upload/readback.
- Several deterministic source schedules preserve deep-hopper character before
  coefficient promotion.

### Material/camera/public lifecycle - next

Only after morphology. Calibrate oxide, metal, environment, fixed camera; then
public auto-start, `Stop`, completion, `Regenerate`, disposal, device loss.

### Performance selection - later

Choose grid/cadence from accepted model. Benchmark only for that decision;
present fidelity/duration/memory/update-rate tradeoffs before equation or
resolution changes.

### Deployment - later

Vite build via stateless Express on Ubuntu EC2 behind trusted HTTPS. No DB,
queue, sessions, auth, or server solver.
